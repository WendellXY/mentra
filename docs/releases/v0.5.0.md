# mentra v0.5.0 / mentra-provider v0.2.0

Mentra `v0.5.0` is a tooling-architecture release. The headline change is a
cleaner boundary between provider-facing contracts and runtime-facing
orchestration, with the provider surface now published as the new
`mentra-provider` crate.

This release also adds a full session lifecycle API, persisted permission
resolution, richer tool progress events, more provider/runtime coverage, and a
large round of test and CI hardening.

## Highlights

### New `mentra-provider` crate

- Extract provider-facing request, response, model, stream, auth, and tool
  contracts into the new `mentra-provider` crate.
- Keep provider serialization concerns separate from Mentra runtime scheduling
  and approval semantics.
- Add Responses transport primitives, including websocket support, HTTP fallback
  controls, metadata event preservation, hosted action contracts, replay, and
  remote compaction support.

### Tooling architecture refactor

- Split provider-facing and runtime-facing tool contracts into explicit layers.
- Introduce dedicated tool orchestration with execution-lane scheduling.
- Refactor builtin, files, and intrinsic tooling into thinner facades with
  focused internal modules.
- Emit richer `ToolProgress` events for shell execution output and file
  operations.

### Session and permission APIs

- Add the `Session` lifecycle surface, session event mapping, hooks, and typed
  session flow support.
- Persist permission rules to SQLite so resumed sessions can keep their approval
  state.
- Support live permission resolution during active sessions.
- Preserve live agent events during turns when sessions are resumed.

### Runtime and provider improvements

- Add transcript compaction and memory summarization plumbing, plus remote
  compaction support in the provider layer.
- Add Ollama and LM Studio support through the OpenAI-compatible Responses API.
- Improve Windows shell execution behavior and strengthen cross-platform CI
  coverage.
- Forward session headers on provider requests where needed.

### Verification and quality

- Expand integration coverage for session lifecycle, runtime tools, compaction,
  resume flows, error recovery, and contract conformance.
- Tighten CI and test behavior across Unix and Windows environments.

## Compatibility and migration

- `ToolDefinition`, `ToolExecutor`, `ToolSpec`, and `ExecutableTool` remain
  available in this release.
- `ToolSpec::builder(...)` remains the supported convenience API for custom
  tools.
- Implement read-only tools with `ToolExecutor::execute(...)`.
- Implement mutating or agent-state-changing tools with
  `ToolExecutor::execute_mut(...)`.
- Provider-visible tool metadata now lives in `mentra-provider`, while runtime
  scheduling and approval metadata stay in `mentra`.

## Publishing order

Publish `mentra-provider 0.2.0` before publishing `mentra 0.5.0`.
